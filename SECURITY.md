# SECURITY.md
## Introduction
The TabGroupSaver-Session-Manager-VSCode-Extension prioritizes the security and integrity of user data. This document outlines our security practices, policies, and procedures to ensure the confidentiality, integrity, and availability of user information.
## Reporting Security Issues
If you discover a security vulnerability in our extension, please report it to us immediately. You can submit a report via our [issue tracker](https://github.com/chirag127/TabGroupSaver-Session-Manager-VSCode-Extension/issues) or by emailing [chirag127](mailto:chirag127@gmail.com).
## Security Audits and Testing
We perform regular security audits and testing to identify and address potential vulnerabilities. Our extension is designed with security in mind, and we follow best practices for secure coding and data handling.
## Data Storage and Handling
Our extension stores user data locally on the user's machine. We do not collect or transmit any user data to our servers or third-party services.
## Dependencies and Libraries
We use the following dependencies and libraries in our extension:
* TypeScript
* VS Code Extension API
We keep our dependencies up-to-date and monitor for any known security vulnerabilities.
## Secure Coding Practices
We follow secure coding practices, including:
* Input validation and sanitization
* Secure data storage and handling
* Regular security audits and testing
## User Responsibilities
Users are responsible for:
* Keeping their VS Code installation and extensions up-to-date
* Using strong passwords and enabling two-factor authentication
* Being cautious when installing extensions from unknown sources
## Contact Information
For any security-related questions or concerns, please contact us at [chirag127](mailto:chirag127@gmail.com).
## Security Policy
Our security policy is designed to protect user data and prevent security breaches. We reserve the right to modify this policy at any time.
